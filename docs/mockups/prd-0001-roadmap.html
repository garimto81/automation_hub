Implementation Roadmap

WSOP Automation Hub v2.0 구현 순서

P1
공유 모듈 완료
  • Hand, RenderInstruction, RenderOutput 모델
  • PostgreSQL 스키마 (init-db.sql)
  • Repository 패턴 (CRUD)
P2
데이터 흐름 통합 진행중
  • automation_ae: InstructionPoller 구현
  • automation_sub: InstructionGenerator 구현
  • automation_feature_table 동기화
  • End-to-End 테스트
P3
안정성 개선 v2.0 핵심 우선
  • Zombie Hunter (Stuck Job 복구)
  • Alembic 도입 (자동 마이그레이션)
  • 대시보드 인증 (Basic Auth)
P4
실시간성 & 확장성 v2.0 확장
  • Redis 큐 도입 (arq, Consumer Group)
  • DB Polling 제거 → 이벤트 기반
  • WebSocket (/ws/dashboard)
  • RenderStatus 확장 (STALE)
P5
모니터링 대시보드 선택
  • 이상 감지 엔진
  • 백그라운드 모니터링 워커
  • React 대시보드 UI
⚡ 참고: Phase 2~4는 부분적으로 병렬 진행 가능합니다. 단, Zombie Hunter (P3)를 우선 구현하세요.
